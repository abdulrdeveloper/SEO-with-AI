## 12. Quick Reference Cheatsheet

### Title Tags
- 50–60 chars | Keyword first | Format: Primary KW – Secondary | Brand

### Meta Descriptions
- 150–155 chars | Include keyword | End with CTA | Unique per page

### URL Slugs
- Hyphens only | Keyword included | 3–5 words | All lowercase | Never change after publish

### Heading Hierarchy
- One H1 | H1 contains primary keyword | H2s for sections | H3s for subsections | Never skip levels

### Core Web Vitals Targets
- LCP < 2.5s | CLS < 0.1 | INP < 200ms

### Content Checklist Per Page
- [ ] Primary keyword in: title, H1, first 100 words, one H2, URL, one image alt
- [ ] 3–5 internal links pointing to this page
- [ ] Word count ≥ competitors' average for this query
- [ ] Author bio (for blog/articles)
- [ ] Schema markup applied

### Keyword Research Tiers
- Head (1–2 words): Brand awareness, high KD — long-term play
- Middle (2–3 words): Balanced, target with pillar pages
- Long-tail (4+ words): Low KD, high conversion, cluster pages

### Anchor Text Distribution
- Branded: 40–50% | Naked URL: 15–20% | Generic: 10–15% | Partial match: 10–15% | Exact match: < 10%

### E-E-A-T Signals Checklist
- [ ] Author bio with credentials on all content
- [ ] First-hand examples and case studies
- [ ] Cited primary sources (studies, official docs)
- [ ] Content updated with current date/data
- [ ] HTTPS + privacy policy + contact info visible

### Algorithm Recovery Quick Triage
```
Traffic drop + update date match → algorithmic
GSC Manual Actions shows issue → manual penalty
Coverage shows mass deindex → technical or Helpful Content
Backlink spike before drop → link spam
```

### The 80/20 of SEO — What Actually Moves Rankings

1. **Publish content matching exact search intent** — format, depth, angle
2. **Get backlinks from real, relevant sites** — even 10 good ones beat 1,000 directories
3. **Fix Core Web Vitals** — especially LCP, which is fixable with image optimization + CDN
4. **Internal linking** — most underused, free, and effective tactic
5. **Update existing content** — refreshing old posts beats publishing new thin ones
6. **Build topical clusters** — depth of coverage beats breadth of topics
7. **E-E-A-T signals** — especially author credentials for YMYL topics
